Fix the Plan Circuit / Itinerary Builder so it correctly 
matches user-entered destination names to the destination 
data and builds an accurate itinerary.

PROBLEM: When users type destination names like "Hampi" 
and "Gokarna", the itinerary shows wrong or unrelated 
locations because the name matching is failing.

FIX 1 — Improve the destination name matching function:

Replace the current key lookup with a fuzzy matching approach:

function findDestinationData(name, destinationData) {
  // Clean the input
  const cleaned = name.toLowerCase()
    .trim()
    .replace(/\s+/g, '')
  
  // Try exact match first
  if (destinationData[cleaned]) {
    return destinationData[cleaned]
  }
  
  // Try partial match — check if any key contains 
  // the input or input contains the key
  const keys = Object.keys(destinationData)
  const partialMatch = keys.find(key => 
    key.includes(cleaned) || cleaned.includes(key)
  )
  
  if (partialMatch) {
    return destinationData[partialMatch]
  }
  
  // Return null if no match — handle separately
  return null
}
