CHANGE 3 — SAVE TO WISHLIST BEHAVIOR
═══════════════════════════════════════

When user taps "Save trip":

function handleSaveToWishlist() {
  // Add to wishlist state
  addToWishlist({
    type: 'itinerary',
    title: itinerary.title,
    days: itinerary.totalDays,
    cost: itinerary.totalEstimatedCost,
    savedAt: new Date().toISOString()
  })
  
  // Show toast notification — slides up from bottom
  showToast({
    message: 'Trip saved to wishlist',
    subtext: 'Get back to it whenever you are ready.',
    icon: '🔖',
    duration: 3000,
    style: 'success'
  })
}

Toast component style:
- Slides up from bottom: translateY(100%) → translateY(0)
- Animation: 250ms ease-out
- Auto-dismisses after 3 seconds with slide-down
- Background: #1A3C5E
- White text, 15px, font-weight 500
- Border-radius: 12px
- Padding: 14px 20px
- Position: fixed, bottom 80px 
  (above bottom nav), left 20px, right 20px
- Two lines:
  Line 1: icon + "Trip saved to wishlist" bold
  Line 2: "Get back to it whenever you are ready." 
  opacity 0.75, 13px
