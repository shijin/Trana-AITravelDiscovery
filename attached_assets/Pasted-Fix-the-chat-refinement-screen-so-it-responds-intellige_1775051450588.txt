Fix the chat refinement screen so it responds intelligently to 
user intent using keyword-based mock responses. 

PROBLEM: Currently the chat returns the same hardcoded response 
regardless of what the user types. When a user says "I want to 
travel to Kerala" it still shows Coorg, Hampi, Pondicherry.

SOLUTION: Build a keyword-intent matching system in the chat handler.

In the chat component, replace the single hardcoded mock response 
with this intent detection function:

function detectIntent(message) {
  const msg = message.toLowerCase();
  
  if (msg.includes('kerala')) {
    return {

      msg.includes('low budget')) {
    return {

  if (msg.includes('beach') || msg.includes('sea') || 
      msg.includes('ocean') || msg.includes('coastal')) {
    return {
      text: "Beach destinations that are less crowded and more your pace:",
      destinations: [

    }
  }

      msg.includes('mountain') || msg.includes('snow') ||
      msg.includes('cool') || msg.includes('cold weather')) {
    return {
      text: "For cooler temperatures and mountain air, these fit your profile:",
      destinations: [
        { id: 1, name: 'Coorg', state: 'Karnataka', 
          reason: 'Misty Western Ghats — cool even in summer, peaceful coffee estates.' },

      msg.includes('anniversary')) {
    return {
      text: "For a romantic trip that does not feel like every other couple holiday:",

    }

  if (msg.includes('quick') || msg.includes('weekend') ||

  }

  // Default fallback for unrecognized intent
  return {
    text: "Let me search for options that better match what you're describing. Here are refined suggestions:",

      { id: 1, name: 'Coorg', state: 'Karnataka',

    ]
  }
}

When the AI responds, render the message text as a ChatBubble first,
then below it render destination suggestion chips in this format:

Each suggestion chip (full-width card, not a small pill):
- White background, border-radius 10px, 
  border 1px solid #E5E7EB, padding 12px
- Left: colored circle with first letter of destination (24px, 
  background #D6F0EF, color #0D7377, font-weight 700)
- Center: destination name (15px bold #1A1A1A) + 
  state (12px gray) on second line +
  reason text (13px italic #666666 on third line)
- Right: chevron right icon (#0D7377)
- Tap: navigate to /destination/:id if id is a number,
  or show toast "Full destination data coming soon" if id is a string
