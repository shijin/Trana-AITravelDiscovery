2. SMOOTH SCREEN TRANSITIONS
═══════════════════════════════════════

Add a global page transition wrapper. Create a component 
called PageTransition that wraps every screen's root element.

Implementation:

Create src/components/PageTransition.jsx:

import { useEffect, useState } from 'react'

export default function PageTransition({ children }) {
  const [visible, setVisible] = useState(false)
  
  useEffect(() => {
    const timer = setTimeout(() => setVisible(true), 20)
    return () => clearTimeout(timer)
  }, [])
  
  return (
    <div
      style={{
        opacity: visible ? 1 : 0,
        transform: visible ? 'translateY(0px)' : 'translateY(12px)',
        transition: 'opacity 280ms ease-out, transform 280ms ease-out',
      }}
    >
      {children}
    </div>
  )
}

Wrap EVERY screen's return statement root div with 
<PageTransition>. This means every route change produces 
a smooth fade-up entrance.

For the QUIZ screen specifically, add a question-level 
transition between questions (not just page-level):

When the user taps Continue and moves to the next question,
animate the outgoing question:
- opacity: 1 → 0, translateX: 0 → -20px, duration: 180ms

Then animate the incoming question:
- Start: opacity: 0, translateX: 20px
- End: opacity: 1, translateX: 0
- Duration: 240ms, ease-out
- Delay: 180ms (starts after outgoing finishes)

Implement using a questionKey state that changes on each 
question advance. Use this as the key prop on the

For the COLORED BACKGROUND section of quiz questions,

transition: background-color 350ms ease-in-out

This means as questions change, the top section smoothly 
transitions between its bold colors rather than cutting.

IMPORTANT CONSTRAINTS
═══════════════════════════════════════

1. Do NOT use any external animation libraries 
   (no Framer Motion, no GSAP, no react-spring).
   Pure CSS transitions and React state only.
   This keeps the bundle size small.

2. All animations must respect 
   prefers-reduced-motion media query:
   @media (prefers-reduced-motion: reduce) {
     * { transition-duration: 0.01ms !important; }
   }

3. Do not change any colors, typography, or spacing.
   Only add transition and animation properties.

4. Do not break any existing navigation or functionality.
   Test that all routes still work after applying animations.

5. The total animation overhead should not slow the app.
   If any animation causes jank, reduce duration by 50%.
